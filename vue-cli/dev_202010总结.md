###### 登录路由跳转处理

```js
场景：从页面A跳转到登录页，关闭登录页或者登录成功再跳转至页面A，登录页有密码登录，验证码登录，找回密码登录。
从页面A跳转到登录页情况有：
	1. 调用某一需登录态的接口，在接口拦截器中跳转至登录页
	2. “我的-登录”入口
想达到的效果：
	1. (前提)页面A点返回则正常返回(this.$router.back())
	2. 登录页点击关闭返回页面A
	3. 点击登录返回页面A
问题：
	如果用 push，则会存在路由返回错乱的情况

// 思路：登录跳转全部用 replace 方法
页面A -> 登录页：replace
登录页 -> 验证码/找回密码：replace
验证码/找回密码 -> 登录页：replace
登录页 -> 页面A：replace
验证码/找回密码 -> 页面A：replace

疑问：如何记录页面A
将页面A的 name 存在 sessionStorage 里
```
